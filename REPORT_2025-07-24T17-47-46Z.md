# Overview of the verl Codebase

## Repository Purpose

**verl** (Volcano Engine Reinforcement Learning for LLMs) is an open-source library for training large language models using various reinforcement learning (RL) algorithms. It supports distributed execution, integration with common model and inference frameworks, and provides recipes and examples for RL-based fine‑tuning, alignment and agentic workflows.

## High Level Layout

- `README.md` – project introduction, key features, news and links to documentation.
- `docs/` – extensive documentation covering algorithms, advanced features, workers, tutorials and FAQ.
- `verl/` – main Python package containing configs, models, training logic and experimental features such as the agent loop.
- `examples/` – runnable scripts/recipes showing how to train with PPO, GRPO and others.
- `recipe/` – curated training recipes reproduced from papers or partner projects.
- `tests/` – unit tests and integration tests, organized by submodule (see `tests/README.md`).
- `scripts/`, `docker/` – helper utilities and Dockerfiles for building environments.

## Key Components

### Package Modules (`verl/`)
- `base_config.py` – dataclass-based configuration with a dict-like interface.
- `protocol.py` – defines `DataProto` for data exchange between components. Includes utilities for padding and merging batches.
- `trainer/` – entry points such as `main_ppo.py` for distributed RL training with Ray, plus configuration templates under `trainer/config`.
- `models/` – wrappers and registries for supported model types (Llama, Qwen, etc.).
- `experimental/agent_loop/` – early-stage support for multi-turn agent workflows running against an asynchronous LLM server.
- `single_controller/` – abstractions for worker groups, resources and distributed execution.

### Examples and Recipes
- `examples/ppo_trainer/README.md` – explains PPO and its configuration knobs.
- Shell scripts under `examples/` demonstrate how to launch training for various models/backends.
- `recipe/README.md` lists open-source projects built on verl, with subfolders containing specific code snapshots to reproduce results (e.g. DAPO, PRIME).

### Documentation Highlights
- `docs/hybrid_flow.rst` – programming guide describing HybridFlow, the design that represents RL pipelines as dataflows.
- `docs/advance/*.rst` – advanced topics such as LoRA support, Ray placement design, and agent loop usage.
- `docs/perf/perf_tuning.rst` – suggestions for maximizing training throughput.

### Tests
- Unit tests are grouped by module (e.g. `tests/trainer`, `tests/models`); see `tests/README.md` for layout.
- Special directories like `tests/special_distributed` contain multi‑GPU or end‑to‑end tests.

## Getting Started
1. Review the [Installation guide](https://verl.readthedocs.io/en/latest/start/install.html) in the documentation.
2. Explore the example scripts in `examples/ppo_trainer` to understand training configuration.
3. Inspect `verl/trainer/main_ppo.py` to see how Ray workers, models and reward functions are orchestrated.
4. Read `docs/hybrid_flow.rst` for the conceptual model behind the API.
5. Run unit tests with `pytest` to validate your environment.

## Suggested Next Steps
- Familiarize yourself with dataclass configs in `verl.base_config` and `verl.trainer.config`.
- Learn how `DataProto` batches are created and manipulated (`verl.protocol`).
- Examine `verl/experimental/agent_loop` if interested in agentic RL or tool integration.
- Check `docs/advance/placement.rst` for details on the Ray worker API and resource management.
- Follow the links in `README.md` and `recipe/README.md` for community projects and further reading.
