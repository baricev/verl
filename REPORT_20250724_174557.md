# Repository Overview

This repository contains **verl**, an open source reinforcement learning (RL) training library for large language models (LLMs). The project originates from ByteDance's Seed team. It implements the HybridFlow framework for efficient RLHF and integrates multiple backends for training and inference.

## Layout

- **verl/** – main Python package with trainers, workers, models and utilities.
- **examples/** – runnable scripts demonstrating various training recipes such as PPO and GRPO.
- **recipe/** – curated end‑to‑end recipes for reproducing research results.
- **tests/** – unit and integration tests organised by feature area; see `tests/README.md` for details.
- **docs/** – Sphinx documentation source. Run `make html` inside `docs` to build locally.
- **scripts/** – helper scripts used in CI and development.
- **docker/** – Dockerfiles and environment setup helpers.

Other key files include `pyproject.toml` (build/ruff config), `.pre-commit-config.yaml` (lint hooks), and `requirements*.txt` files describing dependencies.

## Key Concepts

- **Hybrid programming model** – decouples data flow from computation to support flexible RL algorithms with high throughput.
- **Trainer API** – entry points under `verl/trainer`; e.g. `main_ppo.py` and the `RayPPOTrainer` class drive PPO training loops.
- **Workers** – modules under `verl/workers` that implement actor, critic and reward roles with support for FSDP, Megatron-LM and others.
- **Configurations** – YAML configs in `verl/trainer/config` describe model/backbone, placement and algorithm parameters.

Documentation under `docs/` covers installation, quickstart guides and advanced topics such as FSDP2, vLLM integration and performance tuning.

## Next Steps for New Contributors

1. Read `CONTRIBUTING.md` for contribution workflow, pre-commit setup and testing instructions.
2. Browse the `examples/` directory to understand how training scripts are structured.
3. Review `docs/index.rst` for an overview of available features and pointers to deeper documentation (e.g. `docs/hybrid_flow.rst`).
4. Explore the `verl/trainer` and `verl/workers` packages to see how trainers coordinate workers for RL algorithms.
5. Run `pre-commit run` before committing to ensure lint checks pass.

Happy hacking!
